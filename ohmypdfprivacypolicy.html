01

Introduction

This Privacy Policy describes how Vivek Gaikwad ("we", "us", or "our"), operating under the brand Rise of Rebels, handles information in connection with the Web Scanner Pro Android application ("the App").

We are committed to protecting your privacy. Web Scanner Pro is designed from the ground up to respect your data — all document processing happens entirely on your device, and the App does not collect, store, or transmit any personally identifiable information.

By using Web Scanner Pro, you agree to the practices described in this policy.

02

Information We Collect

Web Scanner Pro does not collect any personally identifiable information. The following summarises exactly what data the App does and does not interact with:

  • No personal data collected — the App does not collect your name, email, phone number, or any personal identifier.
  • No account or registration — you can use the App fully without creating an account or signing in.
  • No file uploads — documents, images, and PDFs you scan are never sent to any server. All processing is performed locally on your device.
  • Crash reports (anonymous) — if the App crashes, Firebase Crashlytics sends an anonymised crash report (device model, OS version, stack trace). No PII is included.
  • Anonymous usage analytics — Firebase Analytics collects anonymous usage events (e.g. which features are used, session length) to help improve the App. No PII is collected or linked.
  • App version check — Firebase Remote Config is used solely to fetch the latest app version number for update notifications. No personal data is involved.
03

How We Use Information

The limited, anonymised data we receive is used exclusively for the following purposes:

  • Diagnosing and fixing technical issues and app crashes
  • Understanding which features are most useful to improve the App
  • Delivering app update notifications via Remote Config
  • Ensuring continued compatibility with new Android versions and devices

We do not use your data for advertising, profiling, or share it with any third party beyond the Firebase services described in Section 4.

04

Third-Party Services (Firebase)

Web Scanner Pro uses the following Google Firebase services. Each service operates under Google's Privacy Policy and processes only anonymised, non-personal data:

💥

Firebase Crashlytics

Collects anonymised crash reports including device model, Android version, and stack trace. No personally identifiable information is included. Used solely to identify and fix app crashes.

📊

Firebase Analytics

Collects anonymous usage events (feature usage, session duration). Data is aggregated and cannot be linked to any individual user. Used to understand app usage patterns and improve features.

⚙️

Firebase Remote Config

Used exclusively to fetch the latest app version number for in-app update notifications. No personal data is transmitted or stored via this service.

All Firebase services are governed by Google's Privacy Policy. You can review it at: https://policies.google.com/privacy and Firebase's data processing terms at https://firebase.google.com/support/privacy.

05

App Permissions

Web Scanner Pro requests only the permissions necessary for its core document scanning functionality. Here is exactly what each permission is used for:

📷

Camera

Used exclusively for document scanning via the in-app camera. Document recognition runs entirely on-device using Google ML Kit. No images are captured without your direct action, and no images are uploaded to any server.

📁

Storage / Files Access

Used only to read files you explicitly select for import and to save PDF documents you create. The App accesses only files you directly choose — it does not browse or index your storage.

No other permissions are requested. The App will always ask for your explicit consent before accessing your camera or storage.

06

Data Storage & Security

All data generated by Web Scanner Pro is stored locally on your device only. There are no backend servers, user databases, or cloud storage involved in the App's core functionality.

  • Scanned PDFs — saved to your device's local storage at the location you choose. Never uploaded to any server.
  • Recent file history — stored in a local Room database on your device. This data never leaves your device.
  • User preferences — theme selection, onboarding status, and other settings are stored locally using Android DataStore. Never synced or transmitted.
  • No cloud backup — the App does not use Google Drive, iCloud, or any cloud backup service for your documents.

Since your files and data are stored only on your device, your security is primarily governed by your device's own security (screen lock, encryption). We recommend keeping your device updated and using a screen lock.
